## 2. Описание реализованной иерархии классов

В качестве базового класса используется класс `Patient` из ЛР-11. Он описывает общие свойства пациента и содержит основные данные и методы для работы с пациентом.

Базовый класс содержит:
- идентификатор пациента;
- имя;
- возраст;
- диагноз;
- специализацию врача;
- температуру;
- артериальное давление;
- статус пациента;
- дату поступления;
- дату выписки;
- историю лечения.

На основе базового класса `Patient` были реализованы производные классы:

### `InPatient`
Класс для стационарного пациента.

Дополнительные атрибуты:
- `room_number` — номер палаты;
- `daily_cost` — стоимость одного дня лечения.

Дополнительный метод:
- `move_to_room()` — перевод пациента в другую палату.

Переопределённые методы:
- `calculate_cost()`
- `process()`
- `get_patient_type()`
- `__str__()`

### `OutPatient`
Класс для амбулаторного пациента.

Дополнительные атрибуты:
- `appointment_date` — дата приёма;
- `consultation_fee` — стоимость консультации.

Дополнительный метод:
- `reschedule_appointment()` — изменение даты приёма.

Переопределённые методы:
- `calculate_cost()`
- `process()`
- `get_patient_type()`
- `__str__()`

### `EmergencyPatient`
Класс для экстренного пациента.

Дополнительные атрибуты:
- `urgency_level` — уровень срочности;
- `ambulance_arrival` — доставлен ли пациент скорой помощью.

Дополнительный метод:
- `increase_urgency()` — повышение срочности.

Переопределённые методы:
- `calculate_cost()`
- `process()`
- `get_patient_type()`
- `__str__()`

### Особенности реализации
В работе были использованы:
- наследование;
- вызов конструктора базового класса через `super()`;
- переопределение методов;
- полиморфизм;
- проверка типов через `isinstance()`;
- единый интерфейс работы с объектами разных типов.

## 3. Демонстрация работы

В файле `demo.py` реализованы сценарии демонстрации работы программы:

1. Создание объектов разных типов:
   - `InPatient`
   - `OutPatient`
   - `EmergencyPatient`

2. Вывод объектов и их основных данных.

3. Демонстрация полиморфизма:
   - вызов `process()` для разных объектов;
   - вызов `calculate_cost()` для разных объектов.

4. Проверка типов с помощью `isinstance()`.

5. Фильтрация объектов по типу:
   - только стационарные пациенты;
   - только амбулаторные пациенты;
   - только экстренные пациенты;
   - только активные пациенты.

6. Использование методов базового и дочерних классов.

7. Сортировка объектов по возрасту.

8. Подсчёт количества записей в истории лечения.
