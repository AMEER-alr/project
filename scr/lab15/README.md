## 2. Реализованные функции и стратегии

### 2.1. Стратегии сортировки (минимум 3)

| Функция | Описание |
|---------|----------|
| `by_patient_age` | Сортировка по возрасту (числовой атрибут) |
| `by_patient_name` | Сортировка по имени (строковый атрибут, алфавитно) |
| `by_patient_status` | Сортировка по статусу (критичные → на лечении → активен → выписан → умер) |
| `by_temperature` | Сортировка по температуре |
| `by_admission_date` | Сортировка по дате поступления |

### 2.2. Функции-фильтры (минимум 2)

| Функция | Описание |
|---------|----------|
| `is_adult` | Только взрослые (≥18 лет) |
| `is_child` | Только дети (<18 лет) |
| `is_critical` | Только в критическом состоянии |
| `is_on_treatment` | Только на лечении |
| `has_fever` | С температурой ≥38°C |

### 2.3. Фабрики функций (замыкания)

| Функция | Описание |
|---------|----------|
| `by_patient_type_filter(type)` | Создаёт фильтр по типу пациента |
| `by_diagnosis_filter(substring)` | Создаёт фильтр по диагнозу |
| `by_age_range_filter(min, max)` | Создаёт фильтр по возрастному диапазону |

### 2.4. Функции для `map()` (преобразование)

| Функция | Описание |
|---------|----------|
| `patient_to_short_string` | Преобразование в короткую строку с эмодзи |
| `patient_to_dict` | Преобразование в словарь |
| `extract_patient_name` | Извлечение имени |

### 2.5. Callable-объекты (паттерн Стратегия)

| Класс | Описание |
|-------|----------|
| `PriorityStrategy` | Определяет приоритет лечения пациента |
| `DischargeCheckStrategy` | Проверяет возможность выписки |

### 2.6. Новые методы коллекции

| Метод | Описание |
|-------|----------|
| `sort_by(key_func, reverse)` | Сортировка по функции-ключу |
| `filter_by(predicate)` | Фильтрация по предикату |
| `apply(func)` | Применение функции ко всем элементам |
| `map_to(func)` | Преобразование в список результатов |
